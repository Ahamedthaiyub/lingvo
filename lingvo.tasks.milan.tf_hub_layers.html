lingvo.tasks.milan.tf_hub_layers._WrapNonLingvoVars(dest_layer: lingvo.core.base_layer.BaseLayer, variables: Collection[tensorflow.python.ops.variables.VariableV1], trainable_variables: Collection[tensorflow.python.ops.variables.VariableV1] = ())[source]¶

Adds variables to the given lingvo layer and appropriate graph collections.

This function helps wrap variables created outside of lingvo so they are correctly handled by lingvo’s trainer and checkpointer. It does the following:

  • makes all variables trackable through dest_layer.vars;

  • ensures variables are in the tf.global_variables() graph collection so the trainer can initialize them;

  • adds the trainable_variables subset to the tf.trainable_variables() graph collection, so they are visible to the learner (i.e. can be trained).

Parameters

  • dest_layer – Lingvo layer to add the variables to.

  • variables – The non-lingvo variables to wrap.

  • trainable_variables – The subset of variables to ensure are trainable.

lingvo.tasks.milan.tf_hub_layers._AddUpdateOpsToSignature(module, signature='default')[source]¶

Creates a copy of a TF1 module signature that triggers stateful update ops.

This helper is intended for TF1-style hub modules (i.e. those loaded via hub.load()) that are to be fine-tuned. It creates a copy of module.signatures[signature] that is functionally equivalent except that it also runs update ops (e.g. updates to batch norm statistics) as a side effect.

(The signatures of TF1 hub modules typically don’t run such ops: they’re often not dependencies of the signature outputs, and are therefore pruned when the module is loaded.)

Parameters

  • module – A TF1-style hub module, in the format returned by hub.load().

  • signature – Name of the signature of module to wrap.

Returns

A callable with the same interface as module.signatures[signature].

class lingvo.tasks.milan.tf_hub_layers.ImageModule(*args, **kwargs)[source]¶

Bases: lingvo.core.base_layer.BaseLayer

Wraps a TF-hub image module, allowing it to be fine-tuned.

NOTE: Input images are assumed to be [0, 1] normalized as per TF-hub standard (see https://www.tensorflow.org/hub/common_signatures/images).

Example usage:

inception_model = ImageModule.Params().Set(
    name='inception_v3',
    module_path=(
        'https://tfhub.dev/google/imagenet/inception_v3/feature_vector/4'),
    update_batch_norm=True).Instantiate()
images = tf.random.uniform([2, 299, 299, 3], minval=0.0, maxval=1.0)
features = inception_model(images)

FProp(_, images)[source]¶

Forward propagation.

The central interface that subclasses should implement. The caller calls FProp with a theta dictionary. E.g.:

foo = InstanceOfASubClassOfFoo(params)
y = foo.FProp(foo.theta, x)

The implementation of FProp() computes a function given the theta and the inputs. E.g.:

subs = self.children
inputs = args[0]
a0 = subs.linear.FProp(theta.linear, inputs)
a1 = subs.softmax.FProp(theta.softmax, a0)
# The same layer applied twice.
a2 = subs.linear.FProp(theta.linear, a1)
return a2
Parameters

  • theta – A NestedMap object containing weights’ values of this layer and its children layers.

  • *args – List args.

  • **kwargs – Keyward args.

class lingvo.tasks.milan.tf_hub_layers.ImageModuleV2(*args, **kwargs)[source]¶

Bases: lingvo.core.base_layer.BaseLayer

Wraps a TF2 hub image module, allowing it to be fine-tuned.

Any regularization losses attached to the hub module are accessible through this object’s losses property.

NOTE: Input images are assumed to be [0, 1] normalized as per TF-hub standard (see https://www.tensorflow.org/hub/common_signatures/images).

Example usage:

params = ImageModuleV2.Params().Set(
    name='efficientnet_b4',
    module_path=(
        'https://tfhub.dev/tensorflow/efficientnet/b4/feature-vector/1'))
model = params.Instantiate()
images = tf.random.uniform([2, 380, 380, 3], minval=0.0, maxval=1.0)
features = model(images)
